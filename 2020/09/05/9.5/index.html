第一道是LOJ分块6

单点插入单点查询

考虑数组查询$O(1)$插入$O(n)$，链表插入$O(1)$，查询$O(n)$

考虑兼顾二者的性质，而块状链表很好的解决了这个问题，但是链表不好写，考虑用动态数组

此题每块内可以放一个动态的数组，每次插入时先找到位置所在的块，再暴力插入，把块内的其它元素直接向后移动一位，当然用链表也是可以的

每根号n次插入后，重新把数列平均分一下块，重构需要的复杂度为O(n)，重构的次数为√n，所以重构的复杂度没有问题，而且保证了每个块的大小相对均衡

  • 查询函数$return \; pr(x, b - 1)$

$b-1$是因为$vector$是从0开始计数的

  • 关于快读快输

   if (! x) putchar('0');

   while (num) putchar(c[num --]);
   putchar('\n');
}

快输没问题

考虑两个版本的快读

read 
inline int read()
{
   int s(0), w(1);
   char ch = getchar();
   while (ch < '0' || ch > '9') {if (ch == '-') w = -1; ch = getchar();}
   while (ch >= '0' && ch <= '9') s = s * 10 + ch - '0', ch = getchar();
   return s * w;
}

第一种,显然好理解，先读入，判断是不是自负并且判负数

read 
LL in()
{
   LL x = 0, f = 1; char ch;
   while(!isdigit(ch = getchar()))(ch == '-') && (f = -f);
   for(x = ch ^ 48; isdigit(ch = getchar()); x = (x << 1) + (x << 3) + (ch ^ 48));
   return x * f;
}

其中$isdigit$判断是不是字母，后边的&&代替了if

第二个for中的$x = (x << 1) + (x << 3)$相当于$x * 10$

其中

$ \because x <<1 =x \times 2,x<<3=x\times 8$

$\therefore (x<<1)+(x<<3) = x \times 2 + x \times 8 = x \times 10$

至此完成转化

第三道是点分治

算上主函数一共十个函数。。。

  • $main$里面初始化的时候$f[rt] = yhm_{size} = n$初始化必须把f包含
  • $getroot$找重心，使之成为$log$

  • $solve$函数$siz[rt] = yhm_size = siz[y]$，所谓的根一直都在更新，并且每次只会进入根（重心）

  • $dfs$函数把当前的小子树遍历一遍，并用一个二元组记录了这个节点到1的距离且来自哪里

  • $cmp$函数把二元组以距离为关键字排序了
  • $work$

首先把自己的儿子们的重心以及他们的大小都遍历出来

二元组多一位是防止边界出锅

然后就是双指针查找

  • $search$函数是在二分查找第一个小与等于$a[i] - dis[l].d$的值

保证是$log$

第四道是树链剖分

以前没有做过，也没有学会

今天先透彻了思想，写了一道模板题目（将近200行…

本质是把树上问题转化为序列问题

然后在序列上跑线段树

考虑四个操作

  • 将树从x到y结点最短路径上所有节点的值都加上z
  • 求树从x到y结点最短路径上所有节点的值之和
  • 将以x为根节点的子树内所有节点值都加上z
  • 求以x为根节点的子树内所有节点值之和

肯定首先要预处理一些东西

跑第一遍$dfs$处理出$dep,son,fa,siz$就是各个节点的深度，自己的重儿子，自己的父亲，自己的节点大小

第二遍$dfs$处理出了重儿子轻儿子的编号，以及各个链的链顶是谁

考虑初始化建立线段树

其次对于第一个和第二个操作

当我们要处理任意两点间路径时，设所在链顶端的深度更深的那个点为x点

  • ans加上x点到x所在链顶端 这一段区间的点权和
  • 把x跳到x所在链顶端的那个点的上面一个点

不停执行这两个步骤，直到两个点处于一条链上，这时再加上此时两个点的区间和即可

然后跑完这个之后，还要比较一下，把最后那条$x,y$同在的链算上答案

其中加权操作与查询操作同理

对于第三个和第四个操作

显然，这一段区间是连续的，考虑左边界为x所在的位置（$id$）

显然，右边界就是$id[x] + siz[x] - 1$，减1是因为$siz[x]$包含了x本身
